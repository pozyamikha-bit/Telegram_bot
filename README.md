# Бот регистрации + приём чеков + админ-панель

Готовый шаблон Telegram-бота на aiogram: регистрирует участника (ФИО,
магазин, телефон), затем принимает от него фото чека/УПД. Все данные
пишутся в Google Таблицу, фото сохраняются в папку `receipts/`. Плюс
встроенная админ-панель для модераторов (см. раздел "Админ-панель"
ниже).

> **Важное изменение:** проект переписан на **aiogram 3.x** (было — 2.x).
> Причина: платформа Amvera при сборке образа игнорирует версию aiogram,
> указанную в `requirements.txt`, и подставляет свою — актуальную 3.x.
> Если бы код остался на 2.x-синтаксисе (`dp.message_handler`,
> `executor.start_polling` и т.п.), он бы успешно собрался, но упал бы
> при запуске с ошибкой импорта, потому что в aiogram 3.x эти вещи
> убраны. Заодно это решило и другую проблему — старая версия aiohttp,
> которую тянула за собой aiogram 2.x, не собиралась на новых версиях
> Python на Windows; в 3.x такой проблемы нет.

## Структура проекта

- `bot.py` — основной файл, логика диалога с пользователем
- `config.py` — загрузка настроек из `.env`
- `google_sheets.py` — функции записи данных в Google Таблицу
- `requirements.txt` — список нужных библиотек
- `.env.example` — образец файла с настройками (токен, ID таблицы)
- `receipts/` — сюда будут скачиваться присланные фото чеков

## Шаг 1. Установка

```
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux

pip install -r requirements.txt
```

## Шаг 2. Токен бота

Если ещё не создавали бота — напишите `@BotFather` в Telegram, отправьте
`/newbot`, задайте имя. Получите токен вида `123456789:AAH...`.

## Шаг 3. Настройка Google Sheets API

Это самая муторная, но одноразовая часть.

1. Зайдите на https://console.cloud.google.com/ и создайте новый проект
   (кнопка выбора проекта сверху -> "New Project").
2. В меню слева откройте "APIs & Services" -> "Library", найдите и
   включите (Enable) два API: **Google Sheets API** и **Google Drive API**.
3. Перейдите в "APIs & Services" -> "Credentials" -> "Create Credentials"
   -> "Service Account". Задайте любое имя, роль можно не выбирать.
4. Откройте созданный сервисный аккаунт -> вкладка "Keys" -> "Add Key" ->
   "Create new key" -> тип **JSON**. Скачается файл — переименуйте его в
   `service_account.json` и положите рядом с `bot.py`.
5. Откройте этот json-файл и скопируйте оттуда поле `client_email`
   (выглядит как `xxxxx@xxxxx.iam.gserviceaccount.com`).
6. Создайте (или откройте) свою Google Таблицу, нажмите "Настройки
   доступа" -> "Добавить" -> вставьте этот email -> дайте права
   **Редактор**. Без этого шага бот не сможет писать в таблицу.
7. Скопируйте ID таблицы из адресной строки браузера — это часть ссылки
   между `/d/` и `/edit`, например для
   `https://docs.google.com/spreadsheets/d/1AbCDefGhIjKl/edit` id это
   `1AbCDefGhIjKl`.

## Шаг 4. Заполнение .env

Скопируйте `.env.example` в файл `.env` и впишите туда токен бота и ID
таблицы, полученные на шагах 2 и 3. Заодно впишите в `ADMIN_IDS` свой
Telegram ID — без этого админ-панель будет недоступна вообще никому
(узнать свой ID можно, написав любому боту-помощнику, например
`@userinfobot` — он в ответ пришлёт число вида `123456789`).

## Шаг 5. Запуск

```
python bot.py
```

Напишите боту `/start` в Telegram и пройдите регистрацию — проверьте,
что в таблице появились строки на вкладках "Регистрация" и "Чеки"
(вкладки создадутся автоматически при первой записи, если их ещё нет).

## Админ-панель

Открывается командой `/admin` — доступна тем, чей Telegram ID указан в
`ADMIN_IDS` ("владельцы"), а также модераторам, добавленным прямо из
самой панели (они хранятся в отдельном листе "Модераторы" в вашей
Google Таблице, не в `.env`).

- **🗓 История** — показывает календарь; при выборе даты выводит список
  всех чеков за этот день (независимо от статуса) в виде кнопок — на
  каждой кнопке имя (или ID, если регистрация не найдена), магазин и
  username отправителя. При выборе конкретного чека — карточка с
  полными данными (ФИО, магазин, телефон) и его фото, а также две
  кнопки: "🗑 Удалить чек" (запросит подтверждение и затем безвозвратно
  удалит строку из листа "Чеки", заодно пробуя удалить и файл фото с
  диска) и "⬅️ Назад к списку" (возвращает к списку чеков за эту же
  дату).
- **🛠 Модерация** — список чеков со статусом "на модерации". У каждого
  чека три кнопки: "✅ Принять" (бот попросит ввести номер купона одним
  сообщением и сам отправит его пользователю), "⚡ Быстрое отклонение"
  (мгновенно отклоняет чек и шлёт пользователю готовый текст — его
  можно поменять в "✏️ Тексты") и "💬 Отклонить с комментарием" (бот
  попросит модератора ввести свой комментарий одним сообщением и
  отправит именно его пользователю).
- **📊 Отчёт** — присылает Excel-файл (.xlsx) с двумя вкладками: все
  чеки (с привязкой к ФИО/магазину/телефону) и все регистрации.
- **👥 Модераторы** (виден только владельцам из `ADMIN_IDS`) — список
  текущих модераторов с кнопкой удаления каждого, плюс кнопка
  добавления нового (владелец присылает Telegram ID нового модератора
  одним сообщением-числом).
- **✏️ Тексты** (виден только владельцам из `ADMIN_IDS`) — позволяет
  менять автоответы бота прямо из чата, без правки кода и повторного
  деплоя: текст кнопки "Правила", просьбу прислать фото, сообщение
  после регистрации, ответ на присланный чек, сообщение при отклонении
  (по умолчанию — "Плохое качество фото чека, просьба повторно
  зарегистрировать чек") и сообщение при принятии чека (в нём можно
  оставить `{coupon}` — вместо этого бот подставит реальный номер
  купона). Все эти тексты хранятся в листе "Тексты" вашей Google
  Таблицы — можно поменять их и вручную прямо там, в столбце "Текст".

Важно про существующие данные: если вы уже запускали более раннюю
версию бота и в листах "Регистрация"/"Чеки" уже есть строки без
заголовка — добавьте вручную строку-заголовок первой строкой в каждый
лист (для "Регистрация": `Дата регистрации, ФИО, Магазин, Телефон,
Telegram ID, Username`; для "Чеки": `Дата, Telegram ID, Username, File
ID, Файл, Статус, Купон, Комментарий`), иначе первая строка с реальными
данными ошибочно будет принята кодом за заголовок.

Если лист "Чеки" у вас уже создан этой же (более новой) версией кода,
но ещё без столбца "Комментарий" (он появился позже остальных) —
допишите вручную "Комментарий" в ячейку с следующей свободной буквой
столбца первой строки, иначе комментарии при отклонении не будут
сохраняться, хотя сам текст пользователю всё равно уйдёт.

Ещё одна мелочь: когда владелец добавляет модератора по Telegram ID
через "👥 Модераторы", бот пытается сам подставить его username — но
это получится, только если этот человек уже хотя бы раз писал боту
раньше (так работает Telegram: по одному ID без предыдущего контакта
имя пользователя не узнать). Если username не заполнился — это не
ошибка и ни на что не влияет, права проверяются по ID, а не по имени.

## Шаг 6. Размещение на сервере (чтобы бот работал постоянно)

Пока вы просто держите `python bot.py` запущенным на своём компьютере,
бот работает только пока не закрыт терминал/компьютер не выключен.
Чтобы бот работал круглосуточно, его нужно перенести на сервер. Два
варианта: обычный VPS (Timeweb Cloud, Selectel, Beget и т.п., где вы
сами всё настраиваете через `systemd`/`screen`/`pm2`), либо готовый
PaaS для ботов вроде Amvera — см. следующий раздел.

## Деплой на Amvera

Amvera сама собирает Docker-образ из вашего git-репозитория, поэтому
здесь есть несколько отличий от обычного локального запуска.

1. **Код должен быть в git-репозитории**, к которому подключена Amvera
   (см. её документацию про подключение GitHub-репозитория —
   `docs.amvera.ru/applications/git/webhooks.html`, для приватных
   репозиториев там нужен Personal Access Token из GitHub).

2. **Секреты и настройки задаются через панель Amvera, а не через
   `.env`** — файл `.env` в git не попадает (он в `.gitignore`), и на
   сервере его взять неоткуда. В разделе "Секреты"/переменные окружения
   вашего проекта на Amvera задайте:
   - `BOT_TOKEN` — токен от BotFather
   - `GOOGLE_SHEET_ID` — ID таблицы
   - `GOOGLE_CREDENTIALS_JSON` — сюда вставляется **всё содержимое**
     файла `service_account.json` целиком, одной строкой (он тоже не
     попадает в git). Код в `google_sheets.py` сам определит, что эта
     переменная задана, и будет использовать её вместо файла.
   - `GOOGLE_SHEET_WORKSHEET_REG`, `GOOGLE_SHEET_WORKSHEET_RECEIPTS` —
     можно не задавать, есть значения по умолчанию.
   - `RECEIPTS_DIR` — поставьте `/data/receipts`, чтобы скачанные фото
     чеков сохранялись в постоянное хранилище Amvera (`persistenceMount:
     /data` из `amvera.yml`) и не терялись при пересборке.
   - `PYTHONUNBUFFERED` — `1`, иначе логи через `print`/`logging` могут
     не показываться в панели.

3. **Файл `amvera.yml`** в проекте — черновой конфиг сборки/запуска.
   Сверьте его с генератором на `manifest.amvera.ru` — Amvera обычно
   создаёт такой файл сама при первом деплое через панель, так что
   ориентируйтесь на то, что показывает она, если наш файл разойдётся
   с интерфейсом.

4. **Гонка при перезапуске**: если после каждого обновления кода бот
   иногда падает с ошибкой вида "Terminated by other getUpdates
   request" — в `bot.py` уже есть небольшая пауза на старте
   (`STARTUP_DELAY_SECONDS`, по умолчанию 3 секунды), это штатная
   рекомендация самой Amvera для таких ситуаций.

## Что стоит доработать под себя

- Сейчас правила акции (кнопка "❓ Правила") — это просто текст-заглушка
  в коде, отредактируйте его под свою акцию.
- Фото чеков сохраняются локально в `receipts/`; если сервер будет часто
  переустанавливаться или нужен доступ к фото из любой точки — можно
  дополнительно настроить загрузку в Google Drive (это отдельная
  небольшая доработка `google_sheets.py`/`bot.py`, спросите, если
  понадобится).
- Проверка "не зарегистрирован ли уже этот пользователь повторно" в
  шаблоне не реализована — сейчас можно пройти регистрацию заново
  много раз. Если нужно ограничение "один пользователь — одна
  регистрация", тоже дорабатывается несложно.
