# Бот регистрации + приём чеков

Готовый шаблон Telegram-бота на aiogram: регистрирует участника (ФИО,
магазин, телефон), затем принимает от него фото чека/УПД. Все данные
пишутся в Google Таблицу, фото сохраняются в папку `receipts/`.

## Структура проекта

- `bot.py` — основной файл, логика диалога с пользователем
- `config.py` — загрузка настроек из `.env`
- `google_sheets.py` — функции записи данных в Google Таблицу
- `requirements.txt` — список нужных библиотек
- `.env.example` — образец файла с настройками (токен, ID таблицы)
- `receipts/` — сюда будут скачиваться присланные фото чеков

## Шаг 1. Установка

```
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux

pip install -r requirements.txt
```

## Шаг 2. Токен бота

Если ещё не создавали бота — напишите `@BotFather` в Telegram, отправьте
`/newbot`, задайте имя. Получите токен вида `123456789:AAH...`.

## Шаг 3. Настройка Google Sheets API

Это самая муторная, но одноразовая часть.

1. Зайдите на https://console.cloud.google.com/ и создайте новый проект
   (кнопка выбора проекта сверху -> "New Project").
2. В меню слева откройте "APIs & Services" -> "Library", найдите и
   включите (Enable) два API: **Google Sheets API** и **Google Drive API**.
3. Перейдите в "APIs & Services" -> "Credentials" -> "Create Credentials"
   -> "Service Account". Задайте любое имя, роль можно не выбирать.
4. Откройте созданный сервисный аккаунт -> вкладка "Keys" -> "Add Key" ->
   "Create new key" -> тип **JSON**. Скачается файл — переименуйте его в
   `service_account.json` и положите рядом с `bot.py`.
5. Откройте этот json-файл и скопируйте оттуда поле `client_email`
   (выглядит как `xxxxx@xxxxx.iam.gserviceaccount.com`).
6. Создайте (или откройте) свою Google Таблицу, нажмите "Настройки
   доступа" -> "Добавить" -> вставьте этот email -> дайте права
   **Редактор**. Без этого шага бот не сможет писать в таблицу.
7. Скопируйте ID таблицы из адресной строки браузера — это часть ссылки
   между `/d/` и `/edit`, например для
   `https://docs.google.com/spreadsheets/d/1AbCDefGhIjKl/edit` id это
   `1AbCDefGhIjKl`.

## Шаг 4. Заполнение .env

Скопируйте `.env.example` в файл `.env` и впишите туда токен бота и ID
таблицы, полученные на шагах 2 и 3.

## Шаг 5. Запуск

```
python bot.py
```

Напишите боту `/start` в Telegram и пройдите регистрацию — проверьте,
что в таблице появились строки на вкладках "Регистрация" и "Чеки"
(вкладки создадутся автоматически при первой записи, если их ещё нет).

## Шаг 6. Размещение на сервере (чтобы бот работал постоянно)

Пока вы просто держите `python bot.py` запущенным на своём компьютере,
бот работает только пока не закрыт терминал/компьютер не выключен.
Чтобы бот работал круглосуточно, его нужно перенести на сервер (VPS) —
например Timeweb Cloud, Selectel, Beget и т.п. (от ~150-300 руб/мес).
На сервере повторяете шаги 1-5, а сам процесс держите живым через
`systemd`, `screen`, `tmux` или `pm2`, чтобы он перезапускался сам при
сбое или перезагрузке сервера.

## Что стоит доработать под себя

- Сейчас правила акции (кнопка "❓ Правила") — это просто текст-заглушка
  в коде, отредактируйте его под свою акцию.
- Фото чеков сохраняются локально в `receipts/`; если сервер будет часто
  переустанавливаться или нужен доступ к фото из любой точки — можно
  дополнительно настроить загрузку в Google Drive (это отдельная
  небольшая доработка `google_sheets.py`/`bot.py`, спросите, если
  понадобится).
- Проверка "не зарегистрирован ли уже этот пользователь повторно" в
  шаблоне не реализована — сейчас можно пройти регистрацию заново
  много раз. Если нужно ограничение "один пользователь — одна
  регистрация", тоже дорабатывается несложно.
